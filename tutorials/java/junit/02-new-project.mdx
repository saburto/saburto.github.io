# 02 - Create a new project

Using [maven archetype quickstart](https://maven.apache.org/archetypes/maven-archetype-quickstart/) to create a new project:

```sh
mvn archetype:generate \
  -DarchetypeGroupId=org.apache.maven.archetypes \
  -DarchetypeArtifactId=maven-archetype-quickstart \
  -DarchetypeVersion=1.5 \
  -DjavaCompilerVersion=21 \
  -DjunitVersion=5.11.4 \
  -DgroupId=com.saburto.tutorial \
  -DartifactId=junit5-tutorial \
  -Dversion=1.0-SNAPSHOT \
  -Dpackage=com.saburto.tutorial.junit5 \
  -DinteractiveMode=false
```

:::info

In order to pass java properties you need to use `-D` before the property name.

:::

For this tutorial we are using the following properties:

|Property|Description| Recommended value |
|--|--|--|
|`javaCompilerVersion`| Java version| 21|
|`junitVersion`| Junit version| 5.11.4|
|`groupId`| Group id of the project | com.saburto.tutorial|
|`artifactId`| Artifact id of the project| junit5-tutorial|
|`version` |Initial version| 1.0-SNAPSHOT|
|`package` |Java package of the project| com.saburto.tutorial.junit5|
|`interactiveMode`| Do not ask for confirmation| false|

As you can see the project is created using the value of the `ArtifactId`, then go to the project directory:

```sh
cd junit5-tutorial
```

Let's see the file structure of the project:

```text
├── pom.xml
└── src
# highlight-next-success
    ├── main
    │   └── java
    │       └── com
    │           └── saburto
    │               └── tutorial
    │                   └── junit
    │                       └── App.java
# highlight-next-success
    └── test
        └── java
            └── com
                └── saburto
                    └── tutorial
                        └── junit
                            └── AppTest.java
```

The `main` folder contains the production code and resources. Under the `test` forlder you will find all the test cases and the code that support the tests.

:::info

Only the  code inside in the main folder is included in the package of the application. The production code can't access to any code in the folder test.

:::


Run the tests:


```sh
mvn test
```

  const [showError, setShowError] = useState(false)
  const [showConfetti, setShowConfetti] = useState(false)

    setShowError(false);
    setShowConfetti(true);
    console.log('yes');
  };

  const onNo = () => {
    setShowError(true);
    setShowConfetti(false);
  };

  return (

      <div>
      <input type="radio" name="quiz" onChange={onYes}></input>
      <label>Yes</label>

      <input type="radio" name="quiz" onChange={onNo}></input>
      <label>No</label>
{showError &&
<p>Check if java and maven is well configured in your terminal session</p>}

{showConfetti && <><p>Congrats! Let's continue</p><ConfettiExplosion /></>}

      </div>


      );

}
